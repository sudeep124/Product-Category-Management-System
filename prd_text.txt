1. Executive Summary
The Simple Product Category System (SPCS) is a focused web application that enables administrators and product managers to create, manage, and organize product categories in a clean, hierarchical structure. This document serves as the single source of truth for all technical, functional, and design decisions.

🎯  Project Goal
Deliver a world-class, production-ready product category management interface that is fast, intuitive, and scalable — with full Azure DevOps integration for sprint-based delivery.

💡  Key Outcome
Any team member should be able to browse, create, edit, and delete product categories within seconds — with zero training required.

3.2 Feature List with User Stories

Feature 1 — Category Dashboard
As an Admin, I want to see a dashboard with total categories, active/inactive counts, and recent changes.
As a Manager, I want a visual overview of the category tree so I can understand the hierarchy at a glance.

Feature 2 — Create Category
As an Admin, I want to create a new category with a name, description, parent (optional), and status.
As an Admin, I want to assign an icon or color to a category to make it visually distinct.
As an Admin, I want the system to prevent duplicate category names at the same level.

Feature 3 — Edit Category
As an Admin, I want to edit a category's name, description, parent, and status.
As an Admin, I want the system to warn me if changing a parent creates a circular reference.

Feature 4 — Delete Category
As an Admin, I want to delete a category, with a confirmation dialog showing the number of subcategories affected.
As an Admin, I want an option to move subcategories to another parent before deleting.

Feature 5 — Search & Filter
As a Manager, I want to search categories by name in real-time (debounced input).
As a Manager, I want to filter categories by status (Active/Inactive/All).
As a Manager, I want to filter by depth level (Root / Level 1 / Level 2).

Feature 6 — Category Tree View
As a Viewer, I want to expand/collapse parent categories to navigate the hierarchy.
As a Viewer, I want drag-and-drop reordering within the same level (optional enhancement).

4.3 Database Schema (PostgreSQL)
🗄️  SQL DDL — Run in your database migration
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

CREATE TYPE category_status AS ENUM ('active', 'inactive');

CREATE TABLE categories (
  id          UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name        VARCHAR(150)    NOT NULL,
  slug        VARCHAR(150)    NOT NULL,
  description TEXT,
  parent_id   UUID            REFERENCES categories(id) ON DELETE SET NULL,
  status      category_status NOT NULL DEFAULT 'active',
  sort_order  INTEGER         DEFAULT 0,
  icon_color  VARCHAR(7),
  created_at  TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
  updated_at  TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
  UNIQUE(name, parent_id),
  UNIQUE(slug)
);

CREATE INDEX idx_categories_parent_id ON categories(parent_id);
CREATE INDEX idx_categories_status ON categories(status);

8.6 Azure Pipelines — CI/CD YAML
📄  azure-pipelines.yml — Save at root of repo
trigger:
  branches:
    include: [main, develop]

pool:
  vmImage: 'ubuntu-latest'

stages:
  - stage: CI
    jobs:
      - job: Build_and_Test
        steps:
          - task: NodeTool@0
            inputs:
              versionSpec: '20.x'

          - script: |
              cd frontend && npm ci && npm run lint && npm test && npm run build
            displayName: 'Frontend: Install, Lint, Test, Build'

          - script: |
              cd backend && npm ci && npm run lint && npm test && npm run build
            displayName: 'Backend: Install, Lint, Test, Build'

          - task: PublishTestResults@2
            inputs:
              testResultsFormat: 'JUnit'
              testResultsFiles: '**/junit.xml'

  - stage: Deploy
    condition: and(succeeded(), eq(variables['Build.SourceBranch'], 'refs/heads/main'))
    jobs:
      - job: Deploy_to_Azure
        steps:
          - task: AzureWebApp@1
            inputs:
              azureSubscription: 'SPCS-ServiceConnection'
              appName: 'spcs-api'
              package: 'backend/dist'
